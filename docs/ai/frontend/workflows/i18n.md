# Workflow: Add or rename an i18n key

Pandame ships a tiny **typed-i18n layer**: keys are stored as JSON
dictionaries under `src/lib/i18n/`, and a Vite-Node script
([`scripts/i18n.ts`](../../../../scripts/i18n.ts)) generates
[`src/lib/types/i18n.d.ts`](../../../../src/lib/types/i18n.d.ts) so that
`$i18n.section.key` is fully typed. CI re-runs the script automatically
on every PR that touches `src/lib/i18n/**` (see
[`.github/workflows/auto-update-i18n.yml`](../../../../.github/workflows/auto-update-i18n.yml)).

## Files

| File                           | What it is                                                                   |
| ------------------------------ | ---------------------------------------------------------------------------- |
| `src/lib/i18n/<locale>.json`   | One JSON per locale. Today: `en.json`.                                       |
| `src/lib/types/i18n.d.ts`      | **Generated**. Do not hand-edit.                                             |
| `src/lib/types/languages.ts`   | Supported locale codes (`'en'` today).                                       |
| `src/lib/stores/i18n.store.ts` | Runtime store + `switchLang` + `init` (locale persisted via `localStorage`). |
| `scripts/i18n.ts`              | The generator. Reads `en.json`, writes `i18n.d.ts`.                          |

## Add a new key

1. **Edit every locale JSON.** Today there's only `en.json`, but treat
   it as the contract — all locales must contain the key. Keys are
   grouped by namespace:

   ```json
   {
   	"core": {
   		"text": {
   			"cancel": "Cancel"
   		}
   	},
   	"notes": {
   		"add": "Add a note"
   	}
   }
   ```

2. **Regenerate the types:**

   ```bash
   npm run i18n
   ```

   This writes `src/lib/types/i18n.d.ts` and prettier-formats it. CI
   does the same on PRs that touched `src/lib/i18n/**`, so don't worry
   if you forget — but commit the regenerated file when you do run it
   locally.

3. **Interpolation.** There's no runtime ICU formatter today. Keys that
   need interpolation use `{placeholder}` markers and are expanded by
   the call site via `String#replace`. The `detail.title` key —
   `"Deal {id}"` — is the canonical example: see
   [`/deals/[deal_id]/+page.svelte`](../../../../src/routes/deals/%5Bdeal_id%5D/+page.svelte)
   for the `replace('{id}', dealId)` pattern. If you need more than
   one placeholder, do them in sequence; if you need pluralisation,
   surface the question first.

4. **HTML in i18n.** Keys whose name ends in `_html` are rendered with
   `{@html}` and bypass Svelte's escaping. The eslint rule
   `svelte/no-at-html-tags` is disabled inline at each call site
   (`<!-- eslint-disable-next-line svelte/no-at-html-tags -->`) — only
   use this for strings you control via the JSON dictionary, never for
   user input.

5. **Use the key** in a component:

   ```svelte
   <script lang="ts">
   	import { i18n } from '$lib/stores/i18n.store';
   </script>

   <button>{$i18n.deals.create_cta}</button>
   ```

6. **Run quality gates** (`npm run quality && npm run check`).

## Add a new locale

1. Add the locale code to
   [`src/lib/types/languages.ts`](../../../../src/lib/types/languages.ts):

   ```ts
   export type Languages = 'en' | 'fr';
   ```

2. Add `src/lib/i18n/<code>.json` with the **same shape** as `en.json`.
   Every key in `en.json` must exist in the new file — the generated
   `I18n` interface treats every key as required.
3. Wire the new locale into
   [`i18n.store.ts`](../../../../src/lib/stores/i18n.store.ts)'s
   `loadLang` switch:

   ```ts
   import fr from '$lib/i18n/fr.json';

   const loadLang = (lang: Languages): Promise<I18n> => {
   	switch (lang) {
   		case 'fr':
   			return Promise.resolve({ lang: 'fr', ...fr });
   		default:
   			return Promise.resolve(enI18n());
   	}
   };
   ```

4. Regenerate types: `npm run i18n` (the generator only reads
   `en.json` — adding a locale doesn't change the type shape).
5. Update [`../reusability.md`](../reusability.md#i18n--libi18n) with
   the new locale.

## Rename a key

1. Search for `$i18n.<old.path>` across `src/` and `e2e/` and update every
   call site.
2. Rename the key in every locale JSON.
3. `npm run i18n` to regenerate types.
4. `npm run check` will catch any missed call site (the typed dictionary
   no longer has the old name).

## Anti-patterns (do not do these)

- Hand-editing `src/lib/types/i18n.d.ts` (regenerated; your edits will be
  lost on the next `npm run i18n`).
- Using a key that exists in `en.json` but not in another locale (or
  vice-versa). Keep all locales in sync.
- Building sentences with `+` concatenation across keys — use a single
  key per sentence, or template literals with named placeholders.
- Hard-coded English copy added "for now" to a component. Prefer adding
  the key first and using `$i18n.*` from the start.
- Forgetting to commit the regenerated `i18n.d.ts` after adding a key
  locally — even though CI auto-commits it, reviewers see a noisier diff.
